# v3.3 Implicit Text Plan

Target release: `3.3.0`
Status: `proposed`
Owner: `Emad`

## Goal

Allow lines without an explicit keyword prefix to be parsed as `text` blocks.

Example:

```it
section: Report Overview
Project: Atlas
Team: Engineering
```

Parses as:

- `section: Report Overview`
- `text: Project: Atlas`
- `text: Team: Engineering`

## Scope (In)

- Parser support for implicit text lines.
- Keep empty-line behavior unchanged.
- Keep special marker behavior unchanged (`---`, table rows, fenced code lines, history markers).
- Diagnostics rule for unknown `keyword:` remains unchanged.

## Scope (Out)

- No shortcut syntax (`#`, `>`, `-`, `!`, `@owner`, etc.).
- No `with:` defaults.
- No multiline grammar changes (`end:` redesign, block state machines).
- No serializer canonical format changes in this release.

## Parsing Rules

1. If line is blank: preserve current blank-line behavior.
2. If line is a known structural marker: parse as current behavior.
3. If line starts with a known `keyword:`: parse as that keyword.
4. If line matches `word:` but `word` is unknown: keep current unknown-keyword handling.
5. Otherwise: parse as implicit `text`.

## Compatibility Policy

- Backward compatible with existing explicit `text:` documents.
- No breaking changes to existing keyword parsing.
- `documentToSource` and Python `to_source` stay as-is for this release.
- Optional pretty-output mode can be discussed after adoption data.

## Test Matrix

- Implicit text basic lines.
- Implicit text mixed with `section:` and `task:`.
- Empty lines preserved exactly.
- Unknown `foo:` still reported/handled as today.
- Divider/table/code/history lines unchanged.
- Trust hash behavior unchanged for explicit source (no serializer rewrite).

## Rollout

1. Implement in Rust parser first.
2. Ensure TS thin layer and Python bridge inherit behavior from Rust parser.
3. Add fixtures covering mixed implicit/explicit documents.
4. Ship in release notes as an additive authoring improvement.

## Exit Criteria

- All existing parser/renderer/validate/trust tests pass.
- New implicit-text tests pass across Rust core and SDK smoke tests.
- No change to package public API signatures.

## Risks

- Accidental capture of lines intended as malformed keywords.
- Divergence between parser behavior and source serializer expectations.

## Mitigations

- Keep unknown `word:` path untouched.
- Keep serializer unchanged in this release.
- Add targeted regression fixtures for ambiguous lines.
